# AI Chat Adapter

## Overview
`ai_chat_adapter` implements the `ai_chat_api.AIInterface` contract and forwards
each call to the autogenerated `ai_chat_service_api_client`. This lets
application code continue to depend on the abstract API while the adapter
speaks to the FastAPI service underneath.

- `AiChatAdapter` translates `generate_response` calls into the request
  payload expected by the service and returns the raw `str|dict` payload the
  service produced.
- `register()` provides a convenience hook that rebinds
  `ai_chat_api.get_ai_interface` to this adapter. You can also pass a custom
  `client_factory` when you need to tweak the generated client (e.g., base URL,
  custom httpx settings).

## Usage
1. Start the AI Chat service (default port 8000, adjust as needed):
   ```bash
   uv run uvicorn ai_chat_service.main:app --reload
   ```
2. Register the adapter in your application and obtain a client:
   ```python
   import ai_chat_adapter
   import ai_chat_api

   ai_chat_adapter.register(base_url="http://127.0.0.1:8000")
   interface = ai_chat_api.get_ai_interface()

   reply = interface.generate_response(user_input="Hello!", system_prompt=None)
   print(reply)
   ```
3. If you need to customize connection parameters (timeouts, headers, etc.),
   pass a `client_factory` that returns a `ServiceClient` configured to your
   needs.

## Testing
- Unit tests live in `src/ai_chat_adapter/tests/`, where monkeypatch fixtures
  stub the autogenerated client to exercise request/response translation.
- Run only this package’s tests:
  ```bash
  uv run pytest src/ai_chat_adapter -q
  ```
- To focus coverage on adapter code and ignore the generated client, scope the
  `--cov` target:
  ```bash
  uv run pytest src/ai_chat_adapter -q \
      -o "addopts=--cov=src/ai_chat_adapter/src/ai_chat_adapter --cov-report=term-missing"
  ```

## Notes
- `src/ai_chat_service_client/src/ai_chat_service_client` is generated from OpenAPI and
  treated purely as a dependency. It is already omitted from mypy and coverage
  in `pyproject.toml`.
- When adding new features that require additional fields from the service,
  regenerate `ai_chat_service_api_client` and update the adapter’s request/
  response mapping accordingly.
