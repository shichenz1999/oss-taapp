# AI Chat Adapter

## Overview
`ai_chat_adapter` implements the `ai_chat_api.AIInterface` contract and forwards each
call to the autogenerated `ai_chat_service_api_client`. This lets application
code continue to depend on the abstract API while the adapter speaks to the
FastAPI service underneath.

- `AiChatServiceAdapter` translates `generate_response` calls into the request
  payload expected by the service and converts the response back into an
  `AIStructuredResponse`.
- `register()` provides a convenience hook that rebinds
  `ai_chat_api.get_ai_interface` to this adapter or to a custom `client_factory`
  (e.g., one that returns an `AuthenticatedClient`).

## Usage
1. Start the AI Chat service (default port 8000, adjust as needed):
   ```bash
   uv run uvicorn ai_chat_service.main:app --reload
   ```
2. Register the adapter in your application and obtain a client:
   ```python
   import ai_chat_adapter
   import ai_chat_api

   ai_chat_adapter.register(base_url="http://127.0.0.1:8000")
   client = ai_chat_api.get_ai_interface()

   reply = client.generate_response(
       user_input="Hello!",
       system_prompt="You are a helpful assistant.",
   )
   print(reply)
   ```
3. If you need custom authentication or connection parameters, provide a
   `client_factory`:
   ```python
   from ai_chat_service_api_client.fast_api_client.client import AuthenticatedClient

   def make_authenticated_client():
       return AuthenticatedClient(base_url="https://service.example.com", token="api-token")

   ai_chat_adapter.register(client_factory=make_authenticated_client)
   ```

## Testing
- Unit tests live in `src/ai_chat_adapter/tests/`, where monkeypatch fixtures
  stub the autogenerated client to exercise request/response translation.
- Run only this package’s tests:
  ```bash
  uv run pytest src/ai_chat_adapter -q
  ```
- To focus coverage on adapter code and ignore the generated client, scope the
  `--cov` target:
  ```bash
  uv run pytest src/ai_chat_adapter -q \
      -o "addopts=--cov=src/ai_chat_adapter/src/ai_chat_adapter --cov-report=term-missing"
  ```

## Notes
- `src/ai_chat_service_api_client/fast_api_client` is generated from OpenAPI and
  treated purely as a dependency. It is already omitted from mypy and coverage
  in `pyproject.toml`.
- When adding new features that require additional fields from the service,
  regenerate `ai_chat_service_api_client` and update the adapter’s request/
  response mapping accordingly.
